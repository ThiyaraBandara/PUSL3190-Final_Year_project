How to Use

Learn how to use our Phishing Detection System effectively.

How to Use

Follow the steps below to run and use the phishing detection tool:

1. Prerequisites

Ensure your system has the following installed:

  • Python 3.7 or higher
  • pip (Python package manager)
  • A terminal or command-line interface (e.g., Command Prompt, Terminal, Bash)
  • Git (optional, if you cloned from a repository)

2. Navigate to the Project Directory

Once the project files are extracted from the ZIP (web_crawler.zip) or cloned from the repository:

cd path/to/web_crawler

Replace path/to/web_crawler with the actual path where you extracted or cloned the project.

3. Set Up a Virtual Environment (Optional but Recommended)

python -m venv venv source venv/bin/activate # On Windows: venv\Scripts\activate

4. Install Dependencies

The project might include a requirements.txt file. Run the following command to install all the required Python libraries:

pip install -r requirements.txt

5. Run the Tool

The core of this phishing detection tool is a command-line program. To start the program, use:

python main.py

If the entry script has a different name (e.g., app.py, cli.py, or similar), replace main.py with the appropriate filename.

6. Input URLs for Analysis

The tool allows you to analyze multiple URLs in one session. Follow these steps:

Step 1: Enter the first URL
Enter the URL to analyze: http://example.com
Step 2: Add more URLs (Optional)

If you want to analyze more URLs, enter them one by one:

Enter another URL (or type 'done' to finish): http://example2.com
Step 3: Start the Analysis

When you have entered all URLs you want to analyze, type 'done':

Enter another URL (or type 'done' to finish): done

After typing 'done', the tool will begin crawling and analyzing all the URLs you've entered.

Note: You can enter as many URLs as you need. The analysis will only begin after you type 'done'.

7. Email Report

After the crawling process is complete, you'll be prompted to enter an email address to receive the analysis report:

Enter email address for the report: your.email@example.com

The system will:

  • Send a detailed report to your email address
  • Save a copy of the report in the local 'reports' directory
  • Display a confirmation message in the terminal
Report has been sent to: your.email@example.com
Report saved locally at: reports/analysis_report_[timestamp].txt

8. Understanding the Report

The analysis report will be delivered to your email and saved locally. The report includes:

  • Detailed analysis of each URL
  • Individual phishing scores for each URL
  • Final classification based on the average phishing score of all analyzed URLs:
    • PHISHING - High risk (Average Score 70+)
    • MAYBE PHISHING - Medium risk (Average Score 31-69)
    • NOT PHISHING - Low risk (Average Score 0-30)
  • Specific reasons for the classification
  • Recommendations for each analyzed URL
Note: The terminal will only show confirmation messages about the report delivery. The actual analysis results are available in the email report and local file. The final classification is determined by calculating the average phishing score across all analyzed URLs.

9. Exit the Program

After finishing analysis, you can exit the CLI by pressing:

Ctrl + C

Or follow any quit command provided within the CLI itself.
